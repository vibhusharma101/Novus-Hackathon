## Findings

### BLOCKER

- `supabase/schema.sql:7` — **`auth.clerk_user_id()` function in a schema the SQL Editor can't write to.** The schema creates `create or replace function auth.clerk_user_id()` — the `auth` schema in Supabase is Supabase-owned and the SQL Editor's `postgres` role cannot CREATE functions there. This was hit as "permission denied for schema auth" in a prior session. All 9 RLS policies reference `auth.clerk_user_id()` and will fail silently or error on apply. Fix: rename the function to `public.clerk_user_id()` and update every policy reference accordingly.

- `src/components/landing/marquee-ticker.tsx:19-26` — **`useEffect` DOM clone appends a new duplicate on every mount.** The effect clones `trackRef.current` and appends it to `parentElement` — but there is no cleanup (`return () => clone.remove()`). In React Strict Mode (default in dev), effects fire twice. In production, any component unmount+remount (route navigation back to `/`) appends another clone, causing 2→3→4 ticker tracks visible simultaneously. The animation reference still targets only the original `ref`, so extra clones are unanimated frozen copies. Fix: return a cleanup function that removes the clone.

### MAJOR

- `src/lib/supabase.ts:18` — **`getToken()` can return `null`; `Bearer null` is a valid-looking but broken auth header.** If Clerk's session is expired or the token call fails, `getToken()` returns `null`. The current code sends `Authorization: Bearer null` to Supabase — TypeScript accepts it (string interpolation coerces null to `"null"`), Supabase rejects it silently as unauthenticated, and the caller gets an RLS-denied empty result with no thrown error. All server actions built on `createUserClient()` will silently return empty data if called with a stale session. Fix: `if (!token) throw new Error('No auth token — user session expired')`.

- `src/components/landing/marquee-ticker.tsx:4` — **Broken singleton: module variable `client` is overwritten on every call.** The `let client` variable is assigned in every `getSupabaseBrowserClient()` call (no early-return guard), so the "singleton" is never actually reused. This means: (a) every subscriber component creates its own Supabase connection, (b) the previous WS connection is orphaned (it remains open, consuming server slots), and (c) realtime subscription events are received by multiple competing callbacks. The 90-second demo depends on exactly one stable realtime connection. Fix: `if (client) return client` before the `createClient` call; token changes should trigger an explicit `client = null` rather than silently re-creating.

- `src/app/globals.css` — **Dark mode token block removed entirely.** The diff deletes the `.dark { ... }` CSS block. shadcn/ui components reference `--primary`, `--card`, `--muted-foreground`, etc. inside their own dark-variant Tailwind classes. Without the dark token values, any user whose OS is in dark mode will see white text on white background (or other broken combinations) on Clerk's sign-in/sign-up pages and the dashboard. The app has no `color-scheme: light` enforcement. Fix: either add back a `.dark { }` block that maps EPRx tokens to dark equivalents, or add `html { color-scheme: light; }` in `:root` to hard-disable dark mode.

- `src/app/page.tsx:265` — **`buttonVariants({ size: 'sm', className: 'mt-6' })` — `className` is not a valid `cva` variant option.** `buttonVariants` is a `class-variance-authority` variant function; it accepts only declared variant keys (`size`, `variant`). Passing `className` inside the options object is silently ignored — the `mt-6` margin never applies. The button sits flush to the paragraph text above it. Fix: `cn(buttonVariants({ size: 'sm' }), 'mt-6')`.

- `src/app/page.tsx` — **Table `key={h}` with `h = ''` (last column header).** The `['Recycler', 'State', ..., ''].map((h) => <th key={h} ...>` uses the empty string `''` as the key for the action column header. React keys must be unique within a sibling list; if a second empty-string header were ever added, React silently renders only one. More importantly, using a semantic value as a key that is expected to be a stable unique string is wrong here — an intentionally blank string key is a code smell. Fix: use `'action'` or use the index for this specific header.

### MINOR

- `src/app/page.tsx:100` — **`text="Certificate in &lt; 5 min"` — HTML entity in a JSX string prop renders literally.** JSX does NOT decode HTML entities in string attribute values (it is not HTML). The `TrustPill` component receives the 12-character string `Certificate in &lt; 5 min` and `{text}` renders it as-is. On screen the user sees `Certificate in &lt; 5 min`. Fix: `text="Certificate in < 5 min"` — raw `<` in a JSX string prop is fine.

Fix the usage to `border-t border-outline-variant pt-3` instead of `hairline-divider pt-3`.
