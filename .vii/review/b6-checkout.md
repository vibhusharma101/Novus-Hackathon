### MAJOR

- `src/components/checkout/checkout-form.tsx:102‚Äď123` ‚ÄĒ **`CtaButton` is a React component defined inside the render function of `CheckoutForm`.** React identifies components by their constructor reference. Every render of `CheckoutForm` allocates a new `CtaButton` function, so React sees a new component type and fully unmounts + remounts the DOM subtree (button) instead of reconciling. While `CtaButton` has no internal state so a visual flash is unlikely in the happy path, it is a React anti-pattern that breaks keys-based reconciliation and will cause the focused button to lose focus on any state update (e.g. typing in the qty field triggers re-render ‚Üí button remounts ‚Üí focus lost). Fix: inline the JSX at both call sites, or lift `CtaButton` outside the module with explicit props (`isPending`, `qty`, `onSubmit`).
