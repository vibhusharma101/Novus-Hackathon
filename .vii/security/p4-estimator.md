## Pass 1 — OWASP Top 10

1. **Broken access control** — `auth()` gates the route. The handler performs **no DB reads** — it only sends the caller's own business description to the model and returns a structured estimate. No tenant data, no IDOR. ✅
2. **Cryptographic failures** — `ANTHROPIC_API_KEY` consumed server-side via the `model` import; never returned to the client. ✅
3. **Injection** — body validated with Zod (`description` 3–2000 chars). The prompt is the user's *own* free-text description fed to the model — there is no secret in the system prompt, no tool, and no privileged action, so prompt manipulation has no security target. Output is constrained by the `estimateSchema` and the three numeric fields are `Math.max(0, Math.round(...))`-clamped server-side. `rationale` is rendered as React-escaped text (no XSS). ✅
4. **Insecure design** — stateless, read-only (no writes). ✅
5. **Security misconfiguration** — generic errors (`400` bad request, `502` estimation failed); no verbose leak. ✅
6. **Vulnerable dependencies** — none added. ✅
7. **Auth/identification failures** — gate present. ✅
8. **Software/data integrity** — `JSON.parse` → Zod `safeParse`; structured-output object re-clamped before return. No untrusted deserialization. ✅
9. **Logging/monitoring failures** — no `console.log` of keys/PII. ✅
10. **SSRF** — only outbound call is to the fixed Anthropic endpoint. ✅
