# Security audit: Phase 4 — Seller Pricing Copilot

_Generated by /vii-cso on 2026-06-13. Base: main._

## Surface
- New endpoints / handlers: `POST /api/suggest-price` (auth-gated + rate-limited; `generateObject` price suggestion grounded in the live per-category market)
- New external calls: Anthropic API via `generateObject` (`claude-sonnet-4-6`)
- New env vars / deps: none

## Pass 1 — OWASP Top 10

1. **Broken access control** — `auth()` gate + `checkRateLimit` first. Reads only **public active-listing prices** for the requested category (`price_per_kg`, via RLS `"listings: read active"`). No tenant data, no IDOR. No recycler-role check (see LOW-1) but the output is public-market advice only. ✅
2. **Cryptographic failures** — `ANTHROPIC_API_KEY` consumed server-side; never returned. ✅
3. **Injection** — body is Zod-validated (`category` enum, `qty_kg` optional non-negative). **No free-text user input reaches the prompt** — the prompt is built from the category label + server-computed market numbers, so there is no prompt-injection surface (unlike the NL estimator). Supabase query is parameterised. ✅
4. **Insecure design** — read-only, stateless. The model's `suggested_price_per_kg` is **clamped server-side** to `[max(0.01, floor*0.5), max(avg*2, floor*1.5)]` and rounded, so a hallucinated/absurd price can't reach the UI. ✅
5. **Security misconfiguration** — generic `400`/`502` errors. ✅
6. **Vulnerable dependencies** — none added. ✅
7. **Auth/identification failures** — gate + shared rate limit present. ✅
8. **Software/data integrity** — output clamped + rounded; input validated. ✅
9. **Logging/monitoring failures** — none. ✅
10. **SSRF** — only the fixed Anthropic endpoint. ✅

## Pass 2 — STRIDE on `/api/suggest-price`

| Threat | Assessment |

### LOW

- `src/app/api/suggest-price/route.ts` — **No recycler-role check.** Any authenticated user (including a buyer) can call this seller tool. Harmless — it reads only public active-listing prices and returns pricing advice; no private data, no writes. Add a recycler-row check only if you want to scope the endpoint to sellers.

## Verdict

**CLEAR** — 0 HIGH, 0 MEDIUM. The cleanest of the AI endpoints: no free-text user input enters the prompt, it's auth-gated + rate-limited, reads only public market data, and clamps the model's price to a sane band before returning. One harmless LOW (missing role scope).
