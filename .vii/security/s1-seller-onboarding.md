## Pass 1 — OWASP Top 10

1. **Broken access control** — `createRecyclerProfile` calls `auth()` first; `clerk_user_id` comes from the session, never the client. RLS `"recyclers: own row"` enforces `clerk_user_id = clerk_user_id()` on insert, so a caller can only create their own row. No IDOR on the write path. ✅
2. **Cryptographic failures** — no secrets/keys; `createUserClient` (anon key + Clerk JWT) only, no `supabaseAdmin`. ✅
3. **Injection** — zod-validated input + parameterised insert. `CPCB_REG_REGEX` = `^[A-Z0-9][A-Z0-9\-/]{3,58}[A-Z0-9]$` — bounded quantifier, no nested repetition → no ReDoS. ✅
4. **Insecure design** — `verified: true` self-asserted (see LOW-1).
5. **Security misconfiguration** — generic error strings; the 23505 branch inspects `error.message` server-side but returns clean copy, no raw DB text to client. ✅
6. **Vulnerable dependencies** — none added. ✅
7. **Auth/identification failures** — page + action both gate on `auth()`. ✅
8. **Software/data integrity** — the document dropzone never uploads or deserializes anything; filename is rendered through JSX (auto-escaped). No file is processed server-side. ✅
9. **Logging/monitoring** — no `console.log` of PII/secrets. ✅
10. **SSRF** — none. ✅

### MEDIUM — RESOLVED in this commit

- `supabase/schema.sql:126` (`"recyclers: public read"`) — **The recycler read policy is row-level only and exposed every column to any authenticated user; this diff added `contact_name` PII to that surface.** The policy `for select using (clerk_user_id() is not null)` combined with the blanket `grant select on all tables to authenticated` meant any signed-in user could run `select clerk_user_id, whatsapp, contact_name, doc_url from recyclers` and read **every** recycler's private fields (`whatsapp` + `clerk_user_id` pre-existing from B5; `contact_name` newly added by S1).
  - **Attack (closed):** Any authenticated buyer queries the `recyclers` base table directly and harvests every recycler's contact name, WhatsApp number, Clerk user ID, and doc URL.
  - **Fix applied:** Added a column-level `GRANT SELECT (id, company_name, state, cpcb_reg_no, capacity_mt, verified) ON recyclers TO anon, authenticated` (after a `REVOKE SELECT ... `) in the ROLE GRANTS section, so the API roles can only read the public subset. `service_role` retains full access for server-side admin reads of a seller's own private fields. Verified non-breaking against all 5 recycler read sites: the two user-client reads (`order-book.tsx` realtime, `certificate/page.tsx` embed) request only public columns; `exchange`/`checkout` use `supabaseAdmin` (service role); `seller.ts` is an insert. No TSX changes required. Owner self-reads of private fields (S2 vault) must use `supabaseAdmin` scoped by `clerk_user_id`.

### LOW

- `src/lib/actions/seller.ts:42` — **`verified: true` is self-asserted.** Any authenticated user can mint a "CPCB VERIFIED" recycler, inheriting the trust badge buyers rely on. Intended for the demo (no real CPCB integration) and set server-side only, but must be replaced with genuine document verification before production.

One LOW remains (`verified: true` self-asserted, intended for demo).
