## Pass 1 — OWASP Top 10

1. **Broken access control** — `auth()` gates the route before any work. Liabilities are read via `createUserClient` (RLS → caller's own brand only). Listings are read via `supabaseAdmin` but constrained to `status='active'` and **public columns only** (`id, category, qty_kg, price_per_kg`, recycler `company_name`/`state`) — the same data the buyer order book already exposes. No cross-tenant leak, no IDOR. ✅
2. **Cryptographic failures** — `ANTHROPIC_API_KEY` is consumed server-side via the `model` import; it is never serialized into the streamed response or sent to the client. ✅
3. **Injection** — SQL is parameterised. The `[[buy:…]]` regex is linear (no catastrophic backtracking), runs on model output bounded by `maxOutputTokens: 800`. Prompt-injection vectors are assessed in LOW-1/LOW-2 — neither reaches a secret or a privileged action. ✅
4. **Insecure design** — Read-only endpoint; no writes, no tools, no state mutation. ✅
5. **Security misconfiguration** — Errors return generic `Unauthorized`/`Bad Request`; the AI SDK masks stream errors by default and the client shows a generic message. No verbose leak. ✅
6. **Vulnerable dependencies** — none added. ✅
7. **Auth/identification failures** — `auth()` gate present. ✅
8. **Software/data integrity** — Request body is `JSON.parse`d then shape-validated with `validateUIMessages`; no untrusted deserialization. ✅
9. **Logging/monitoring failures** — no `console.log` of keys/PII. ✅
10. **SSRF** — the only outbound fetch is to the fixed Anthropic endpoint; no user-controlled URL. ✅

## Findings

### HIGH
_None._

### MEDIUM — RESOLVED (commit after de74e74)

> A shared Postgres-backed per-user rate limiter (`checkRateLimit`, 30 req/60s) was added in front of `/api/copilot`, `/api/estimate-liability`, and `/api/chat`. See `.vii/security/p4-rate-limit.md`. The finding below is retained for history.

- `src/app/api/copilot/route.ts` — **No per-user rate limit or per-message input cap on a paid LLM endpoint.** Messages are capped at 40 by count, but individual message text length is unbounded and there is no throttle. An authenticated buyer can script repeated/large `/api/copilot` calls, each costing Anthropic tokens + a DB round-trip — a financial-DoS / quota-exhaustion vector. (The pre-existing `/api/chat` has the same gap.)
  - **Attack:** authenticated user loops large requests against `/api/copilot`, burning the org's Anthropic budget and rate limits.
  - **Fix:** add a per-user rate limit (e.g. Upstash Ratelimit or equivalent) and a total-input-token guard before calling `streamText`. Acceptable for the single-user demo; should land before any public deployment. Not a one-line change (needs a limiter), so documented rather than auto-fixed.

### LOW

- `src/app/api/copilot/route.ts` — **Second-order prompt injection via seller-controlled `recycler.company_name`** — _partially hardened in this commit._ A malicious seller could set their company name to injection text, interpolated into the buyer's copilot system prompt inside the "LIVE ACTIVE LISTINGS" block. Impact is bounded to *biased advice* (no tools/secrets/actions). **Mitigation applied:** a `sanitize()` helper strips newlines and `[` `]` from `company_name`/`state` before interpolation, so the value can't break out of its data line or inject a fake `[[buy:]]` marker. Residual risk (in-line persuasion text) is low and inherent to mixing untrusted data into prompts.

- `src/app/api/copilot/route.ts` — **First-order prompt injection via user messages** can only steer the model to off-topic text; no security impact (no tool execution, no secret in context, no state change). The `[[buy:<id>]]` deep-link is independently re-validated by the checkout page, so a hallucinated/forged id harmlessly redirects to the exchange. Noted for completeness.

## Verdict

**FIX-MEDIUM-RECOMMENDED** — 0 HIGH, 1 MEDIUM. The endpoint is access-control-clean and leaks no cross-tenant data; the MEDIUM is an unthrottled-LLM cost/DoS gap (shared with the pre-existing `/api/chat`) that needs a real rate limiter before production — acceptable for the single-user demo. Two LOW prompt-injection notes, neither reaching data or actions.
