## Findings

### HIGH

- `src/app/api/chat/route.ts:4-14` — **Unauthenticated AI endpoint.** `POST /api/chat` has no auth check. Any anonymous caller — including bots — can drive unlimited requests to Anthropic and run up your API bill. The Clerk middleware matcher includes `/(api|trpc)(.*)` but middleware alone does not protect API routes unless the handler explicitly calls `auth.protect()` or checks `userId`. Attack: `curl -X POST https://yourapp.com/api/chat -d '{"messages":[{"role":"user","content":"..."}]}'` in a loop from anywhere. Fix: add `const { userId } = await auth(); if (!userId) return new Response('Unauthorized', { status: 401 })` at the top of the handler.

- `src/lib/supabase.ts:1-9` — **Service role key used for all database access, bypassing RLS.** The single shared Supabase client uses `SUPABASE_SERVICE_ROLE_KEY`, which has superuser-level database access and ignores all Row Level Security policies. Any forgotten `where user_id = ?` clause in any future query leaks every user's data to every other user. Attack: once any data endpoint is added without an explicit user filter, an authenticated attacker can read all rows. Fix: create a per-request client using the user's JWT for RLS-scoped queries; only use the service role client for privileged background operations.

### MEDIUM

- **`@clerk/ui` pulls in Solana wallet adapter deps with 15 moderate CVEs.** `npm audit` reports 15 moderate severity vulnerabilities all rooted in `@solana/web3.js` → `@solana/wallet-adapter-*` → `@clerk/ui`. These are transitive and likely not reachable in a Next.js auth flow, but they bloat the dependency tree significantly and increase supply chain risk. Fix: pin `@clerk/ui` to `0.3.24` (the last version before the Solana adapter was introduced) or switch to Clerk's built-in appearance props without `@clerk/ui`.

- `src/app/api/chat/route.ts:5` — **No input validation or body size limit on chat messages.** A caller can send an arbitrarily large `messages` array, triggering an expensive Anthropic API call or causing an OOM error. Fix: validate message shape with zod and enforce a max message count/length before passing to `streamText`.

### LOW

- `src/app/(dashboard)/error.tsx:14` — **Full Error objects logged to browser console in production.** `console.error(error)` in a client component exposes stack traces and potentially sensitive filenames/paths to end users via DevTools. Use a proper error reporting service and log only a sanitized message to the console.

- `.env.local` — **Real Clerk API keys written to disk by `clerk init`.** Confirmed `.gitignore` excludes `.env*`, so these will not be committed. No action needed — just ensure this pattern holds as the project grows and never commit a `.env.production` with real secrets.

- `src/middleware.ts` — **Public routes include bare `/`.** The landing page is intentionally public, which is correct. Just note that if any server action or API route is called from the landing page in the future it must include its own auth check.
