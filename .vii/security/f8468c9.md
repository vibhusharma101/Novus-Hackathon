### MEDIUM

- `src/lib/actions/buyer.ts:62` — **`saveLiabilities` does not cap the `items` array length.** An authenticated user can POST 10,000 `{category:'rigid', market_kg:1}` entries; the loop and `rows.map` run 10,000 iterations before the upsert sends an oversized payload to Supabase. There are only 3 valid categories, so a 1-line guard closes this completely.
  - **Attack:** Authenticated buyer sends `items` with thousands of duplicate-but-valid entries, causing excessive server CPU + DB load per request.
  - **Fix:** Add `if (items.length > PLASTIC_CATEGORIES.length) return { ok: false, error: 'Invalid liability data.' }` immediately after the empty-array check.
