## Findings

### HIGH

- `src/app/layout.tsx:47` (commit `4ee2715`) — Hardcoded Pendo API key `a2dcdf6d-60fc-4424-ac01-5565c1ce4b10` committed to git history. Attack: anyone with repo access (or public GitHub) can use this key to inject events into the Pendo account or exhaust quota. Fix: **key moved to `NEXT_PUBLIC_PENDO_API_KEY` env var** (this PR) — key must also be rotated in Pendo dashboard as it is permanently in `git log`.

### MEDIUM

- `src/components/copilot/copilot-panel.tsx:10` — `const AGENT_ID = 'HJBcjrkMod2_D5jL1_7hVjzPgiE'` hardcoded. Attack: ID is in git history; someone could spoof agent attribution events in Pendo. Fix: move to `process.env.NEXT_PUBLIC_PENDO_AGENT_ID`.

- `src/components/pendo-initializer.tsx:34–50` — PII (email, phone, WhatsApp number) sent to third-party Pendo analytics in `pendo.identify()` without explicit user consent notice. Under India's DPDP Act 2023 and GDPR, personal data requires disclosure and legal basis. Attack surface: PII leaks to Pendo if any data breach occurs on Pendo's side. Fix: strip personal contact fields from the `visitor` payload; retain only pseudonymous identifiers (`userId`, `accountId`, `companyName`, `state`).

- `src/components/seller/order-handshake.tsx` — `buyer_company_name` sent in `order_accepted` / `order_declined` Pendo events on the seller side. Attack: a compromised Pendo account would expose counterparty business identity across all seller sessions. Fix: replace with `buyer_id` hash or omit entirely.

- `src/components/checkout/checkout-form.tsx` — Exact transaction financials (`credits_cost`, `platform_fee`, `total`, `recycler_name`) sent to Pendo. Attack: third-party analytics has full transaction ledger. Fix: send only categorical data (`category`, `qty_kg`) — drop exact amounts and recycler name.

### LOW

- `src/global.d.ts:1` — `declare var pendo: any` overrides the narrower types in `types/pendo.d.ts`, silencing all type errors on Pendo calls. Fix: delete `global.d.ts`; consolidate to one typed declaration.

- `src/pendo.d.ts` vs `src/types/pendo.d.ts` — Three separate files declare conflicting shapes for the `pendo` global. TypeScript resolves them non-deterministically. Fix: single canonical declaration in `src/types/pendo.d.ts`.

- `src/components/pendo-initializer.tsx:13` — `pendo.initialize(...)` called unconditionally on mount. If the CDN script fails to load (NEXT_PUBLIC_PENDO_API_KEY not set, network error, ad-blocker), this throws a `ReferenceError` that crashes the component tree. Fix: guard with `typeof pendo !== 'undefined'`.

- `src/components/copilot/copilot-panel.tsx:95` — Logic bug: `is_suggestion_click: SUGGESTIONS.includes(text)` uses the raw untrimed `text` state instead of the `t = text.trim()` local. Suggestion clicks that match won't be flagged if the user has leading/trailing whitespace. Fix: use `t`.

- `src/app/layout.tsx` — Pendo CDN script loaded without Subresource Integrity (SRI) hash. A compromised `cdn.pendo.io` could serve malicious JS. Fix (hardening): add a `integrity` attribute once Pendo publishes a pinned hash, or self-host the bundle.
